# Module & Project Structure — boilerplate-python

## Project Source Layout

```
src/
  pythonboilerplate/
    __init__.py
    _version.py        # Generated by setuptools-scm
    cli.py             # CLI entry point (argparse / click)
tests/
  test_cli.py
  pythonboilerplate/
    test_*.py
docs/
  conf.py              # Sphinx configuration
  _static/
  _templates/
```

## Package Configuration

- Package manager: **UV** with PEP 735 `[dependency-groups]`
- Version: managed by **setuptools-scm** (from git tags)
- Build backend: `setuptools` + `setuptools-scm`
- Binary build: **Nuitka** (`mise run build`)

## Key Files

| File               | Purpose                                          |
| ------------------ | ------------------------------------------------ |
| `pyproject.toml`   | All project metadata, dependencies, tool configs |
| `mise.toml`        | Tool versions (`[tools]`) + task includes        |
| `.mise/tasks.toml` | All `mise run <task>` definitions                |
| `uv.lock`          | Locked dependency versions (commit this)         |
| `dprint.jsonc`     | Formatter config for TOML/YAML/JSON/Markdown     |

## Import Style

- Standard library imports first, then third-party, then local
- Absolute imports preferred over relative
- All public APIs must have type annotations
